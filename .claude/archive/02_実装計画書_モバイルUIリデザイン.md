## 1. 全体方針

### 1.1 段階的アプローチ
リファクタは**4 Phase に分割**し、各 Phase 完了時にユーザー確認 → マージ可能な単位とする。

- Phase 0: 共有コンポーネント新設（`PillTabBar` / `MobileScreenHeader`）
- Phase 1: `MobileLayout.tsx` を Pill化（最も視覚インパクト大、リスク小）
- Phase 2: 各画面のヘッダー / 大見出し統一
- Phase 3: カレンダーセルの密度調整・FAB配置調整・最終調整

### 1.2 ブランチ戦略
```
refactor/web-first-v2 (現行マイグレーションSSOT)
  └── feature/mobile-ui-redesign (この作業のローカル feature branch)
       ├── Phase 0 commit
       ├── Phase 1 commit
       ├── Phase 2 commit
       └── Phase 3 commit
```

migration ブランチ上で進めるため、各 Phase は**コンフリクト最小単位**に分けてコミットする。

## 3. Phase 別計画

### 3.1 Phase 0: 共有コンポーネント新設（推定 2-3h）

#### 3.1.1 `PillTabBar.tsx`
浮遊型ボトムナビとサブタブの両方で使える汎用 Pill タブバー。

**API設計**:
```typescript
interface PillTabBarProps<T extends string> {
  tabs: Array<{ id: T; label: string; icon?: LucideIcon }>;
  activeId: T;
  onChange: (id: T) => void;
  variant?: "floating" | "inline";       // floating: 下部浮遊 / inline: コンテナ内
  size?: "sm" | "md" | "lg";
  className?: string;
}
```

**スタイル仕様**:
- `floating`: `fixed bottom-[calc(env(safe-area-inset-bottom)+16px)]`、`rounded-full`、`shadow-lg`、`bg-notion-bg`
- アクティブ: `bg-notion-accent text-white` の pill 背景
- 非アクティブ: `text-notion-text-secondary`、`hover:bg-notion-hover`
- 44pt タップ領域確保

**配置**: `frontend/src/components/Mobile/shared/PillTabBar.tsx`

#### 3.1.2 `MobileScreenHeader.tsx`
各タブのメインヘッダー（大見出し + サブテキスト + 右端アクション/サブタブ）。

**API設計**:
```typescript
interface MobileScreenHeaderProps {
  title: string;              // 大見出し（28-32px bold）
  subtitle?: string;          // サブテキスト（13-14px secondary）
  right?: ReactNode;          // 大見出し行の右側に配置（PillTabBar / アクションボタン）
  sticky?: boolean;           // スクロール時に縮小して固定
}
```

**レイアウト**:
```tsx
<div className="flex items-end justify-between gap-3 px-5 pt-2 pb-4">
  <div className="min-w-0 flex-1">
    <h1 className="text-[30px] font-extrabold tracking-tight leading-tight">{title}</h1>
    {subtitle && <p className="mt-1 text-[14px] text-notion-text-secondary">{subtitle}</p>}
  </div>
  {right && <div className="shrink-0">{right}</div>}
</div>
```

これで「大見出し左 + サブタブ右」のFS-11 / FG-10 要件を満たす。

#### 3.1.3 ユニットテスト
- `PillTabBar.test.tsx`: アクティブ切替、コールバック、a11y属性
- `MobileScreenHeader.test.tsx`: title/subtitle 表示

### 3.2 Phase 1: `MobileLayout.tsx` Pill化（推定 2h）

#### 3.2.1 変更点
1. `<header>` から `border-b` を削除、`Life Editor` 固定文字列を削除
2. ヘッダーは **Menu(左) + UndoRedoButtons(右)** の2要素のみに簡素化
3. `title` プロップは廃止（大見出しはコンテンツ側 `<MobileScreenHeader>` で表現する）
4. ボトムタブを **`<PillTabBar variant="floating">`** に置換
5. メインコンテンツの `padding-bottom` をボトムナビ分（約 80px）確保

#### 3.2.2 Undo / Redo の disabled 表示
現状の `MobileLayout.tsx` は `{undoDomains.length > 0 && <UndoRedoButtons />}` で条件レンダリング → 完全に消える。新仕様では **常時表示し、無効時は disabled スタイル** に変更:
```tsx
<UndoRedoButtons
  domains={undoDomains}
  alwaysVisible    // ← 新規 prop、空配列なら opacity-35 + cursor-not-allowed
/>
```
理由: ヘッダー右側の位置が画面ごとにずれると視覚的に不安定。常時同じ位置に置く。

#### 3.2.2 Before → After 抜粋
```tsx
// Before: 罫線付き、画面端まで張り出し
<nav className="flex shrink-0 border-t border-notion-border bg-notion-bg pb-[env(safe-area-inset-bottom)]">

// After: 浮遊 Pill
<PillTabBar
  variant="floating"
  tabs={tabs}
  activeId={activeTab}
  onChange={onTabChange}
/>
```

#### 3.2.3 影響: `MobileApp.tsx`
```tsx
<MobileLayout
  activeTab={history.current}
  onTabChange={handleTabChange}
  onOpenDrawer={handleOpenDrawer}
  title={getTabTitle(history.current)} // ← 追加
>
```

### 3.4 Phase 3: カレンダー密度 + FAB + サイドバー + 最終調整（推定 4h）

#### 3.4.1 カレンダーセル（CalendarGrid 均等幅固定）
- 高さ `min-h-[78px]` → **`min-h-[72px]`** に微調整
- `MAX_CHIPS_PER_CELL = 3` → **`1`**
- `+N more` を**件数バッジ**（右上に `bg-notion-accent/15 text-notion-accent rounded-full px-1.5 text-[10px]`）に変更
- セル罫線を `border-notion-border` → `border-notion-border/40` に淡化

**重要 (固定幅クリップ仕様)**: 現状の `grid-cols-7` は Tailwind では `repeat(7, minmax(0, 1fr))` に展開されるが、**子要素にも `min-w-0` が必須**。`MobileMonthlyCalendar.tsx` の月グリッドを以下のように修正:

```tsx
// 月グリッド本体
<div className="grid grid-cols-7 ...">
  {calendarDays.map(...)}
</div>

// DayCell（変更点）
<button
  className="relative flex min-h-[72px] w-full min-w-0 max-w-full
             flex-col overflow-hidden border-b border-r
             border-notion-border/40 px-[3px] pb-[3px] pt-1"
  style={{ boxSizing: 'border-box' }}
>
  ...
  {/* チップ: clip 切り捨て (省略記号なし、折り返しなし) */}
  <span className="block w-full min-w-0 max-w-full truncate-clip
                   whitespace-nowrap overflow-hidden text-[10px]
                   leading-tight px-1 py-0.5 rounded">
    {item.title}
  </span>
</button>
```

**Tailwind カスタムユーティリティ** (`frontend/src/index.css` に追加):
```css
/* "..." なしで物理的に切る省略 (text-overflow: clip) */
.truncate-clip {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: clip;
}
```

理由: Tailwind 標準の `truncate` は `text-overflow: ellipsis` を含むため "..." が表示される。今回の要件 (FS-08) では「省略記号なしで途中切り捨て」が必須のため、独自ユーティリティを定義する。

#### 3.4.2 FAB再配置 ★決定済み: Pillナビ中央統合（2026-05-16）
- 現状: `absolute right-4 z-40 ... bottom-{fabBottomClass}` のレイヤー分離 FAB
- 新仕様: `<PillTabBar>` 内に `centerSlot` プロップを追加し、`+` ボタンを中央に挿入
- `PillTabBarProps` 拡張:
  ```typescript
  interface PillTabBarProps<T extends string> {
    tabs: Array<{ id: T; label: string; icon?: LucideIcon }>;
    activeId: T;
    onChange: (id: T) => void;
    variant?: "floating" | "inline";
    size?: "sm" | "md" | "lg";
    centerSlot?: ReactNode;            // ← FAB / 検索ボタンなどを中央に挿入
    className?: string;
  }
  ```
- 既存 FAB ロジック（`MobileCalendarView` 内の `<button class="fab">`）を削除し、`MobileLayout` で `centerSlot={<FabButton onClick={onCreateAction} />}` を渡す
- `onCreateAction` はタブごとに異なる動作を担うため、画面側からコールバックで注入（Schedule: 新規予定、Materials: 新規ノート、Work: タスク選択、Settings: なし→無効化）

#### 3.4.3 左ドロワー（サイドバー）リファイン
対象ファイル: `frontend/src/components/Mobile/shared/MobileLeftDrawer.tsx` および `frontend/src/MobileApp.tsx` の `renderDrawerSection()`

**現状からの変更点**:
| 現状 | 新仕様 |
|---|---|
| 罫線アンダーラインのタブ切替 (`border-b-2 border-notion-accent`) | **Pill セグメント** (`PillTabBar variant="inline" size="sm"`) |
| ヘッダーが「Menu」固定 | ノート/デイリーの選択に応じてヘッダーを動的に切替（「ノート」/「デイリー」） |
| 検索ボックスなし | 上部に検索フィールドを追加（プレースホルダーのみで OK、実検索は別タスク） |
| 「新しいノート」ボタンが Drawer 内ボタンに埋もれ | 上部固定の **CTA ボタン**として独立配置（`bg-notion-accent text-white rounded-xl`） |
| アクティブノードの強調が弱い | `bg-notion-accent/15` + `text-notion-accent font-semibold` で明示 |
| フッターの SidebarLinks がリストの一部 | `border-t` で軽く区切った独立フッター領域 |

**レイアウト構造**:
```tsx
<MobileLeftDrawer isOpen={...} onClose={...}>
  <div className="flex h-full flex-col">
    {/* Header: 大見出し + Close */}
    <DrawerHeader title={drawerTitle} onClose={handleCloseDrawer} />

    {/* Pill タブ */}
    <PillTabBar
      variant="inline" size="sm"
      tabs={[{id:"notes", label:"Notes", icon: StickyNote}, {id:"daily", label:"Daily", icon: BookOpen}]}
      activeId={subTab}
      onChange={setSubTab}
      className="mx-4 mb-3"
    />

    {/* 検索 + 新規作成ボタン (固定) */}
    <DrawerSearchInput className="mx-4 mb-2" />
    <button className="mx-4 mb-2 px-3.5 py-2.5 bg-notion-accent text-white rounded-xl ...">
      <Plus size={16} /> 新しいノート
    </button>

    {/* ツリー (スクロール領域) */}
    <div className="flex-1 min-h-0 overflow-y-auto px-2 pb-4">
      {subTab === "notes" ? <MaterialsTree ... /> : <DailyTree ... />}
    </div>

    {/* フッター: SidebarLinks (固定) */}
    {sidebarLinks.length > 0 && (
      <div className="shrink-0 border-t border-notion-border px-2 py-2">
        ...
      </div>
    )}
  </div>
</MobileLeftDrawer>
```

**重要な実装ポイント**:
- ヘッダー・タブ・検索・新規ボタン・フッターはすべて `flex-shrink: 0`
- スクロールは中央のツリー領域のみ (`flex-1 min-h-0 overflow-y-auto`)
- ノート名は「...」省略 OK（カレンダーチップとは違いツリーは省略記号があってよい）
- `useEdgeSwipeBack` との競合回避: Drawer が開いているとき (`isDrawerOpen`) は edge swipe back を `isBlocked` で無効化 (現状で実装済み)

## 4. リスクと緩和策

| リスク | 影響度 | 確率 | 緩和策 |
|---|---|---|---|
| 浮遊ナビがコンテンツを隠す | 中 | 中 | main 領域に `pb-[88px]` を追加、最下部要素確認 |
| Tailwind `notion-*` トークンと違うクラス使用 | 高 | 低 | `coding-principles.md §5` を再読、PR レビュー時にgrep確認 |
| Provider 順序を誤って壊す | 高 | 低 | UI のみ変更しContextの呼び出し順は触らない |
| Migration ブランチとのコンフリクト | 中 | 中 | Phase ごとに小さくコミット、まめに rebase |
| Edge swipe back が浮遊ナビと干渉 | 低 | 低 | `useEdgeSwipeBack` の `isBlocked` でナビ Pill エリアを除外 |
| iOS Capacitor 化への影響 | 中 | 低 | 純粋な Tailwind/React 変更のみ。WebView 依存ロジック触らず |
| 既存テストが落ちる | 低 | 中 | snapshot は再生成、振る舞いテストは構造変更に追従 |

## 9. 学習メモ（ユーザー向け）

ユーザーは Anthropic 提供技術の学習途中とのことなので、本リファクタで触れる概念を要約：

| 概念 | このプロジェクトでの登場 | 学びポイント |
|---|---|---|
| **Skill** | このセッションで `pdf/docx/xlsx` 等が利用可能 | Claude が「特定の文書形式を作る」専用の手順書。会話開始時に Claude が読みに行く |
| **Plugin** | `.claude/CLAUDE.md` 参照のように、CLAUDE.md がClaude Code のコンテキストとして自動読込される | プロジェクト固有の規約・前提を Claude に毎回伝える仕組み |
| **MCP Server** | life-editor 本体に `mcp-server/` がある（30ツール）。今回のリファクタとは別物 | Claude が外部システム（DB/API/ファイル）を操作するための共通プロトコル |
| **Cowork mode** | 現在使っている UI。ファイル作成・ブラウザ操作などができる | Claude Code を「コーディング以外の作業」にも広げたモード |

実装フェーズに入る前に、必要であれば各概念を別途解説する。
