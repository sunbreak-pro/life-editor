## 0. Context（なぜやるか）

要件定義書 §1 のとおり、モバイルUIのトライ&エラーが Tauri/Electron ビルドとデータ層に絡まって遅い。**純粋 Web の Vite 単体環境**を `prototype/` に分離し、3つの Artifacts TSX を「独立起動 → 統合プレビュー」で実環境確認できるようにする。

- **non-goals**: 本番 `frontend/` への変更 / データ層再現 / Tauri・Electron・Capacitor バンドリング / 自動テスト整備（要件定義 §1.3）
- **制約**: 1ファイルTSX維持 / Catppuccin Mocha 固定 / Provider・Context 不使用 / 永続化なし / 依存最小（要件定義 §3.3, NFR-2）

## 1. Phase 0: 生 Artifacts の隔離（本セッションで完了）

Artifacts 原本TSX 3点を `.claude/docs/vision/plans/`（書類棚）から `prototype/_artifacts/`（凍結原本）へ移し、Phase 2 移植元を確定する。

- [x] `prototype/_artifacts/` を作成し、3 TSX を移動
- [x] 原本は以後改変しない（移植は作業コピーで行う）

**理由**: `.claude/` は本番ビルド対象外かつ「完了プランは archive 送り」運用のため、動く部品の置き場として不適。原本を凍結しておくと、移植後の差分照合と不変要件マニフェスト（unified demo 冒頭 MV-1〜SIDEBAR-1）の検証基準になる。

## 2. Phase 1: Vite + React + Tailwind v3 初期セットアップ

### Steps

- [ ] `prototype/` にブランチ独立の Vite プロジェクトを構築（本番 `frontend/` とは別 `package.json` / `node_modules`）
- [ ] 依存を要件定義 NFR-2 の許可リストに限定（`@tauri-apps/*` / `electron` / `@capacitor/*` / Supabase / SQLite / MCP を一切入れない）
- [ ] Tailwind を **v3 構文**で設定（v4 構文混入を防ぐ）
- [ ] `vite.config.ts` に `server: { host: true }` を入れ、iPhone 実機（同一Wi-Fi）からも確認可能にする
- [ ] HashRouter を採用（ファイル直開きでも動く。要件定義 §11 オープン課題への回答案）

### 想定コマンド

```bash
git switch refactor/web-first-v2 && git switch -c prototype/mobile-ui
mkdir -p prototype && cd prototype
npm init -y
npm i react react-dom react-router-dom lucide-react
npm i -D vite @vitejs/plugin-react typescript @types/react @types/react-dom \
        tailwindcss@^3 postcss autoprefixer
npx tailwindcss init -p          # v3 系の tailwind.config.js / postcss.config.js を生成
```

### tailwind.config.js（v3 構文・抜粋）

```js
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: ["./index.html", "./src/**/*.{ts,tsx}"],
  theme: { extend: {} },
  plugins: [],
};
```

### vite.config.ts（抜粋）

```ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
  server: { host: true, port: 5173 }, // host:true で LAN 公開 → iPhone Safari 確認
});
```

- [ ] `cd prototype && npm run dev` が 1 秒以内に起動
- [ ] `http://localhost:5173/` が表示される
- [ ] Vite 起動ログに `Network: http://192.168.x.x:5173/` が出る（iPhone 確認用）
- [ ] `package.json` に禁止依存が無い（`grep -E "tauri|electron|capacitor|supabase" prototype/package.json` が空）

### 配色方式の統一（要件定義との差分解消）
