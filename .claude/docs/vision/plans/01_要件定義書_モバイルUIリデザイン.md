## 1. 背景と目的

### 1.1 背景
現状の Life Editor モバイル版は、Desktop の情報密度をそのまま縮小した設計で、以下の課題がある。

- ヘッダー・サブタブ・下部タブが**すべて罫線で区切られた密集レイアウト**で、コンテンツ閲覧領域が狭い
- 下部タブが画面端まで張り出した「板」状で、参考メモアプリのような**浮遊感や指の押しやすさ**に欠ける
- タイポグラフィのコントラストが弱く、画面に入った瞬間「何の画面か」が瞬時に分かりにくい
- カレンダーセルが密で、参考カレンダーアプリ（Material風）のような**空白の心地よさ**が失われている

### 1.2 目的
参考画像2点（メモアプリの大胆な余白×太字、Material系カレンダーの密度バランスとPill型ボトムバー）の設計言語を Life Editor のモバイル版に統合し、以下を達成する。

1. **コンテンツ最大化**: ヘッダー・ナビのクロームを最小化し、本文・カレンダーへの没入を高める
2. **片手操作性**: 下部Pillナビと FAB を再配置し、親指リーチ範囲に主要アクションを集約
3. **タイポグラフィ駆動**: 太字大見出し + 軽い本文の二段構成で、各画面の役割を瞬時に伝える
4. **デザイントークン一貫**: 既存の `notion-*` Tailwind トークン上で実装し、デスクトップ版と矛盾しない

### 1.3 非目標 (Non-Goals)
- デスクトップUIの変更
- 機能追加（カレンダーへの新規 view 追加、Notes機能拡張など）
- 配色テーマ自体の刷新（既存ライト/ダークトークンを尊重）
- アクセシビリティ要件の引き下げ（タップ領域は44pt以上を維持）

## 3. デザイン原則

### 3.1 5つの設計原則
1. **Whitespace as UI**: 余白そのものを情報構造として使う。`px-5 py-6` を基本とし、コンテナ間は `gap-6` 以上を確保。
2. **Pill, Float, Shadow**: 主要ナビ・FAB・サブタブを**浮遊型 Pill** に統一。下部Safe Area から `16-20px` 浮かせ、`shadow-lg` で輪郭。
3. **Typography Hierarchy**: 画面の役割を**1段目=月/日付/タイトル（28-32px bold）**、2段目=サブ情報（13-14px medium）、3段目=本文（15px regular）の3階層で固定。
4. **Quiet Chrome**: ヘッダーは罫線ゼロ。背景同色で溶け込ませ、アイコン2つまで（Menu と Today/Action）に絞る。
5. **One Color Accent**: アクセント色は1画面1用途に限定（FAB、今日マーカー、アクティブタブのいずれか1つ）。色の連打を避ける。

### 3.2 採用要素（参考画像から）
- メモアプリの「太字大見出し + 軽い本文」のタイポリズム
- メモアプリの「下部浮遊Pillバー」（back / forward / search / + / tabs / menu）
- カレンダーアプリの「セル内にバッジ1つだけ」の控えめな密度
- カレンダーアプリの下部「月/3日/検索/ToDo/+」セグメント

### 3.3 不採用要素（参考画像にあるが取らない）
- Pokémon GO 等の広告枠（不要）
- ブラウザ的な戻る/進むボタン（Edge swipe back で代替済み）
- ステータスバー風の上部メタ情報

## 6. 必要な要素 / 不要な要素マトリクス

### 6.1 必要（参考画像から取り込む）
- 浮遊型 Pill ボトムナビ（角丸 + shadow + 浮かせる）
- ヘッダーの罫線除去
- 大見出し（28-32px bold）の徹底
- カレンダーセルの「1チップ + 件数バッジ」化
- セクション間の `gap-6` 以上の余白
- 浮遊型サブタブセグメント

### 6.2 不要（現状から削減 or 退ける）
- ヘッダーの `border-b` 罫線
- 下部ナビの `border-t` 罫線と端まで張り出すレイアウト
- カレンダーセルの「3チップ + +N more」（情報密度過多）
- サブタブのアンダーライン式アクティブ表現
- 「Life Editor」の固定タイトル文字列（タブごとの文脈に置換）
- 設定画面の罫線多用 → 余白で区切る

### 6.3 据え置き（変更しない）
- 4タブ構成（Schedule / Work / Materials / Settings）
- 左ドロワーの Notes/Daily 切替
- FAB の存在と機能（位置は調整）
- 既存の Provider 構造・DataService 抽象
- カレンダーの操作モデル（タップで sheet、長押しで DnD）

## 8. オープン課題と決定事項

### 8.1 決定済み（2026-05-16 ユーザー合意）
- ✅ **FAB配置 (Phase 3)**: Pillナビ中央に統合 → **将来 Phase 5 でナビから削除しスライドパネルに移行**（後述）
- ✅ **進め方**: レビュー → フィードバック反映 → Phase 0 着手
- ✅ **右上アイコン**: Schedule のカレンダーアイコンは廃止し **Undo / Redo の2ボタン**に統一
- ✅ **サブタブ位置**: 月/週/日 は大見出しの **右側**に小型 Pill セグメント
- ✅ **セル形状**: `aspect-ratio: 1 / 1.3` で縦をやや長めに。日付数字は **左上**配置、チップは下に **左寄せ**
- ✅ **セル内容**: 日付 + アイテム名(1個)のみ、右上バッジ廃止
- ✅ **カレンダースクロール**: 縦スクロール廃止、1画面に固定
- ✅ **セルタップ UX**: 下からスライドアップするパネルで当日アイテム一覧 + 追加ボタン

### 8.2 オープン課題（実装着手時に再確認）
1. **下部Pillナビの幅**: 中央寄せ可変幅（デモ準拠） vs 4タブ均等幅
2. **大見出しのスクロール挙動**: collapse / sticky / 固定の3案
3. **Materials画面のドロワー左タブ（Daily/Notes）**: 現状維持 vs 上部 Pill セグメントに昇格（→ Phase 5 のタブ別 Sidebar 設計で再検討）
4. **スライドパネルの下方向スワイプ閉じる動作**: gesture detect の実装方法（`useTouchSwipe` を新設するか）

## 9. 将来計画（Future Roadmap）— Phase 5 以降

ユーザーが 2026-05-16 に新たに表明した方針。**今回のリファクタ（Phase 0-3）には含まれない**が、後続フェーズで実装する。

### 9.1 タブ別 Sidebar コンテンツ切替（FR-01）

現状: 左ドロワーの内容は Materials タブ専用（Notes/Daily ツリー）で、他タブ（Schedule/Work/Settings）からは「No additional content」の空状態が出る。

新仕様: **アクティブな下部タブに応じて Sidebar の内容を動的に切り替える**。

| アクティブタブ | Sidebar コンテンツ |
|---|---|
| Schedule | フィルタ（タスク/予定/ルーチン on-off）、検索ボックス、Dayflow View へのジャンプ、カレンダー設定（週始まり/祝日表示） |
| Work | タイマー履歴、Pomodoro プリセット一覧、対象タスク選択、累計セッション集計 |
| Materials | 現状維持（Notes/Daily ツリー） |
| Settings | Sidebar 不要 → ドロワーを開いてもメインと同じ「設定」を表示するか、ドロワーアイコン自体を非表示にする |

実装方針:
- `MobileLeftDrawer` を `<DrawerHost activeTab={tab}>` に変えて、子コンポーネントを `tab` で切り替える
- 各タブ用の Sidebar コンポーネントを新設:
  - `ScheduleSidebar.tsx`
  - `WorkSidebar.tsx`
  - 既存 `MaterialsSidebar.tsx`（Drawer 内ラッパー）
  - Settings は ドロワーアイコン非表示 or 軽量ヘルプ表示

### 9.2 ナビ中央 `+` ボタン削除 → スライドパネルへ移行（FR-02）

現状（Phase 3 完了時点）: 下部 Pill ナビ中央に `+` ボタン（FAB 統合済み）。タブごとに新規作成ダイアログを開く。

新仕様: **中央 `+` をナビから削除し、4タブ均等構成に戻す**。新規作成はコンテキスト依存のエントリポイントに移す:

| タブ | 新規作成の入口 |
|---|---|
| Schedule | カレンダーセル → スライドアップパネル内の「この日に予定を追加」CTA ボタン（既に Phase 3 で実装済み） |
