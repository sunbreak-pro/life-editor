# 指示書（毎回 Claude に渡すマニュアル）

あなたは life-editor リポジトリで作業するエンジニアです。
このファイルの内容が、毎回の作業開始時に渡されます。短く・確実に・1歩ずつ進めてください。

## 今回やること

1. `scripts/loop-engine/TODO.md` を開く。
2. 未完（`- [ ]`）の項目を **上から1つだけ** 選ぶ。
   - 未完が無ければ「DONE: タスクなし」とだけ出力して、何も変更せず終了する。
3. 選んだ項目を実装する。
4. 実装できたら、`TODO.md` のその行を `- [ ]` から `- [x]` に書き換える。

## 厳守ルール（Loop Engineering のガードレール）

- **1回の実行で扱うのは1項目だけ**。複数まとめてやらない（小さく・元に戻せる範囲で）。
- 終わったとき、型チェック `cd frontend && npm run build` とテスト `npm test` が**通る状態**にする。
- 確信が持てないときは、推測で大きな変更をしない。`TODO.md` に「要確認:」のメモを足して止める。
- 既存の設計規約（`.claude/CLAUDE.md` と `.claude/rules/`）に従う。
- `.session-branch` や git のブランチ運用（§7.4）には触らない。

## 出力

- 何をしたかを2〜3行で要約して終わる。
