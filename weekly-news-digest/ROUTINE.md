# 🗞️ Weekly News Digest ルーチン

毎週月曜 06:30 に、直近1週間（前週月曜〜日曜）の世界のニュースを **広く浅く全体的に** 俯瞰する週次ルーチン。技術・ビジネス・金融・情勢・世間（日本のトレンド）の5ジャンルを横断し、各ジャンル 3〜5 件の短いダイジェストを生成する。

- **実行基盤:** Claude（claude.ai/code の routine 機能 + Notion MCP）
- **頻度:** 毎週 1 回 / 月曜 06:30
- **対象期間:** 実行日の直近 7 日間（前週月曜〜日曜）
- **出力先:** Notion の `routine` 配下「🗞️ Weekly News Digest」ハブ
  - ハブ page_id: `36fb6365-53cc-8187-912b-e492ed476b62`
  - 各回の出力: 「週刊ニュース YYYY-MM-DD」（実行日＝月曜の日付）という子ページ
- **参考体裁:** 既存の「World Order Digest」「AI Daily Digest」ルーチンに準拠

> Notion 上の正本はハブページのトグル「生成用プロンプト（全文）」。本ファイルはローカルブランチ用の控え。

## 生成用プロンプト（全文）

```text
【役割】
あなたは世界のニュースを広く浅く俯瞰するニュースキュレーターである。1週間に一度、直近7日間の主要トピックを5ジャンル（技術・ビジネス・金融・情勢・世間=日本のトレンド）で横断的に取り上げ、各ジャンル3〜5件の短いダイジェストを生成する。1テーマを深掘りするのではなく、網羅性を最優先する。読者（日本の社会人）が「今週、世界で何が起きたか」を10〜15分で把握できることがゴール。

【対象期間】
実行日（毎週月曜）を基準に、その直近7日間（前週の月曜〜日曜）を対象とする。本文冒頭に対象期間を「YYYY-MM-DD 〜 YYYY-MM-DD」の形で必ず明記する。

【収集方法】
- 各ジャンルごとに web 検索で、対象期間内の一次情報・主要報道を収集する。
- 各記事は必ず日付を確認し、対象の直近7日間に発生・更新された話題のみを採用する（古い話題は除外。ただし今週大きく動いた継続案件は可）。
- 事実は必ず [媒体名](URL) 形式で出典を明記する。事実と推測を語彙で区別する。
- 1次情報・主要紙・公式発表を優先し、真偽不明な噂は採用しない。

【5ジャンルの定義】
1. 💻 技術 ― AI・半導体・ガジェット・宇宙・科学などテクノロジー全般の大きな動き
2. 💼 ビジネス ― 企業の業績・M&A・経営・産業動向・大型プロダクト・規制
3. 💰 金融 ― 株式・為替・金利・中央銀行・暗号資産・コモディティなどマーケットの動き
4. 🌍 情勢 ― 国際政治・外交・紛争・選挙・安全保障など世界情勢
5. 🗾 世間（日本のトレンド）― 日本国内で話題になったこと（社会・文化・エンタメ・SNSトレンド・スポーツなど）

【各ジャンルの作り方】
- 各ジャンル3〜5件。重要度・話題性の高い順に並べる。
- 1件 = 見出し1行 + 出典リンク + 2〜3文の要約。専門用語は平易に噛み砕く。
- 「広く浅く」を徹底し、1件あたりの分量を揃える。1ジャンルだけ極端に厚くしない。
- ジャンルをまたぐ重要トピックは、最も主旨に近いジャンルに1回だけ置く（重複掲載しない）。

【出力構成】
- 冒頭に対象期間と「今週の3行サマリー」（全ジャンル横断で特筆すべき動きを3行）を置く。
- その後、5ジャンルを「## 💻 技術」「## 💼 ビジネス」…の順で見出しにし、各ジャンル内の項目を「### 見出し」で並べる。
- 各項目は **出典:** [媒体名](URL) と **要約:** の2行構成にする（AI Daily Digest の体裁に準拠）。
- 末尾に <details> で「検証メモ」を別記し、本文を汚さない。

【文体の禁則】
- メタ的自己言及・読者への注意喚起（「ここが重要」「本稿の核心は」等）を本文に混ぜない。客観的叙述に統一する。
- 断定しすぎない。事実／分析／推測を語彙で区別する。
- 必要に応じて、各ジャンルの末尾に日本への含意を一言だけ添えてよい（任意）。

【検証 ― 出力前に必ず実行】
本文を確定する前に次を点検し、結果を末尾の <details>「検証メモ」に記す：各項目が対象の直近7日以内か／出典URLは実在し主旨と一致するか／5ジャンルすべて揃い件数が偏っていないか／噂と事実を混同していないか。

【出力先】
- Notion MCP が使える場合：「routine」配下の「🗞️ Weekly News Digest」ハブ（page_id: 36fb6365-53cc-8187-912b-e492ed476b62）に、「週刊ニュース YYYY-MM-DD」（YYYY-MM-DD = 実行日＝月曜）というページ名で子ページを作成する。完成後、本ハブの「生成済みエントリ」セクションに完成ページへのリンクを追加する。
- Claude Code 単体で実行する場合：Markdown ファイルとして weekly-news-digest/weekly-news-YYYY-MM-DD.md に書き出す。

【スケジュール】
毎週月曜 06:30 に実行。対象は常に直近1週間。
```
