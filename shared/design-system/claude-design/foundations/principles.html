Creation Principles — Life Editor Design System

作成原則の要約 · 正本 = shared/design-system/PRINCIPLES.md / CLAUDE.md §6-7

不変式トップ6

  1. 色は必ずトークン経由。notion-* のみ・hex 直書き禁止
  2. 主要コンテナ背景に透明度を使わない（不透明トークン）
  3. 新規 UI は shared/src/components/ に集約
  4. i18n は props 経由・en/ja 両 catalog
  5. DataService はコールバック注入
  6. 本文は WCAG AA ≥ 4.5:1

カラーの 4 役割

  • Chrome（地） bg/text/border — テーマ可変
  • Brand/Accent cobalt 主＋mint 第2 — 可変
  • Semantic success/danger — 可変
  • Functional/Data status band・chart — テーマ固定（状態/データの符号化）
主アクション/主選択 = cobalt。mint は差し色（chip/tag/positive）。

透明度ポリシー

  • 本体背景は不透明トークンで塗る
  • 許容: hover / backdrop / accent 薄塗り / border・ring / disabled / shadow
  • ✕ 未定義 bg-notion-* は透明落ち
  • ✕ ポップオーバー本体に bg-*/70・backdrop-blur

ライト / ダーク

  • dark は pure #000 を使わない（#16161a）
  • light も pure #fff 回避（#fafafa）
  • accent はダークで明度↑（#5b8cff）
  • dark の on-accent は near-black #0a1024（白だと不可読）

新しいトークンを足す手順

  • :root に light 生値 → [data-theme="dark"] に dark 値 → @theme に --color-notion-* マッピング
  • 部品からは bg-notion-* / text-notion-* で参照 · コントラスト検証（本文・on-accent は AA）
  • PR で tokens.css と PRINCIPLES.md §3.3 を同時更新（色の SSOT を 2 箇所で食い違わせない）
